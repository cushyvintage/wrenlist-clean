SECTION 9: TROUBLESHOOTING REFERENCE
================================================================================

If deployment fails:
  1. Check SQL syntax in Supabase SQL Editor error messages
  2. Verify Supabase project is accessible
  3. Look for existing table conflicts
  4. Check browser console for CORS errors

If authentication fails:
  1. Verify .env.local credentials are correct
  2. Check Supabase email provider is configured
  3. Look at Auth logs in Supabase Dashboard
  4. Clear browser localStorage and cookies

If API returns 403:
  1. Verify user is authenticated (check browser DevTools)
  2. Check RLS policies are correct
  3. Verify auth.uid() is available
  4. Check Supabase logs for policy violations

If forms don't submit:
  1. Open Network tab in DevTools
  2. Check API response for errors
  3. Verify RLS allows INSERT
  4. Check form validation rules

See QUICK_DEPLOY_GUIDE.md for detailed troubleshooting.
